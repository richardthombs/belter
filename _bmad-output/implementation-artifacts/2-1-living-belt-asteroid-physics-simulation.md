# Story 2.1: Living Belt — Asteroid Physics Simulation

Status: ready-for-dev

## Story

As a **player**,
I want to see asteroids moving, drifting, and colliding in the belt around me,
so that the world feels alive and dynamic rather than static.

## Acceptance Criteria

1. **Given** the `SimulationLoop.cs` tick, **when** running, **then** each asteroid's position is updated each tick according to its velocity vector (Newtonian drift — FR8).

2. **Given** two asteroids on a collision course, **when** they meet (circle overlap: `distance < r1 + r2`), **then** collision is detected, both asteroids change trajectory based on mass/momentum (elastic collision with `mass = radius²` proxy), and fragments may be generated (FR8).

3. **Given** a collision where at least one asteroid has `radius > 20f`, **when** it occurs, **then** the smaller asteroid is marked `IsDestroyed = true`, child fragment entities are created in the database with new trajectories, and the parent wreckage does not appear in subsequent `WorldStateUpdate` broadcasts.

4. **Given** `AsteroidManager.cs`, **when** updating asteroid state each tick, **then** all trajectory updates are server-authoritative and reflected in the next `WorldStateUpdate` to all connected clients — clients receive `velocityX` and `velocityY` in `AsteroidSnapshot` for dead-reckoning between ticks.

5. **Given** the starting sector is generated by `SectorGenerator`, **when** spawned, **then** each asteroid has a small random initial drift velocity (2–12 units/s in a random direction) — the belt is never static from the first tick.

- [ ] Task 4 — Implement `CollisionResolver` (AC: 2, 3)
  - [ ] Replace the stub in `server/BelterLife.Simulation/Physics/CollisionResolver.cs` with the full implementation below.
  - [ ] **Algorithm summary:**
    - Iterate all pairs `(i, j)` where `i < j`; skip if either `a.IsDestroyed`
    - Detect overlap: `distance(a1, a2) < a1.Radius + a2.Radius`
    - Guard against "already separating": compute relative velocity along collision normal; if `dvn >= 0` (moving apart), skip
    - Apply elastic collision response (see Dev Notes for formula)
    - Fragmentation rule: if `a1.Radius >= 20f || a2.Radius >= 20f`, the smaller asteroid is marked `IsDestroyed = true` and 1–2 fragments are created (see Dev Notes)
    - Return `List<Asteroid>` of all newly created fragment entities (not yet in DB; `Id = 0`)
  - [ ] Full implementation:
    ```csharp
    using BelterLife.Shared.Entities;

    namespace BelterLife.Simulation.Physics;

    /// <summary>
    /// Detects and resolves asteroid-asteroid collisions each tick.
    /// O(n²) pair iteration — acceptable for 20–50 asteroids per sector.
    /// </summary>
    public class CollisionResolver
    {
        private readonly Random _rng = new();

        /// <summary>
        /// Resolves all collisions in <paramref name="asteroids"/> for one tick.
        /// Mutates velocity (and IsDestroyed) on colliding asteroids in-place.
        /// </summary>
        /// <returns>Newly created fragment asteroids (Id = 0; caller must persist to DB).</returns>
        public List<Asteroid> Resolve(List<Asteroid> asteroids)
        {
            var fragments = new List<Asteroid>();

            for (int i = 0; i < asteroids.Count - 1; i++)
            {
                var a1 = asteroids[i];
                if (a1.IsDestroyed) continue;

                for (int j = i + 1; j < asteroids.Count; j++)
                {
                    var a2 = asteroids[j];
                    if (a2.IsDestroyed) continue;

                    float dx = a1.X - a2.X;
                    float dy = a1.Y - a2.Y;
                    float distSq = dx * dx + dy * dy;
                    float minDist = a1.Radius + a2.Radius;

                    if (distSq >= minDist * minDist) continue; // no contact

                    float dist = MathF.Sqrt(distSq);
                    if (dist < 1e-6f) continue; // degenerate: same centre

                    float nx = dx / dist; // collision normal from a2 → a1
                    float ny = dy / dist;

                    float dvx = a1.VelocityX - a2.VelocityX;
                    float dvy = a1.VelocityY - a2.VelocityY;
                    float dvn = dvx * nx + dvy * ny;

                    if (dvn >= 0f) continue; // already separating — skip

                    // Elastic collision impulse: j = 2·dvn / (1/m1 + 1/m2)
                    float m1 = a1.Radius * a1.Radius;
                    float m2 = a2.Radius * a2.Radius;
                    float impulse = 2f * dvn / (1f / m1 + 1f / m2);

                    a1.VelocityX -= impulse / m1 * nx;
                    a1.VelocityY -= impulse / m1 * ny;
                    a2.VelocityX += impulse / m2 * nx;
                    a2.VelocityY += impulse / m2 * ny;

                    // Fragmentation — only when a large asteroid is involved
                    if (a1.Radius >= 20f || a2.Radius >= 20f)
                    {
                        var smaller = a1.Radius <= a2.Radius ? a1 : a2;
                        smaller.IsDestroyed = true;
                        int count = _rng.Next(1, 3); // 1 or 2 fragments
                        for (int f = 0; f < count; f++)
                        {
                            float angle = (float)(_rng.NextDouble() * Math.PI * 2);
                            float scatter = 15f + (float)(_rng.NextDouble() * 20f);
                            float fRadius = MathF.Max(3f, smaller.Radius * 0.4f);
                            fragments.Add(new Asteroid
                            {
                                SectorId = smaller.SectorId,
                                X = smaller.X + MathF.Cos(angle) * scatter,
                                Y = smaller.Y + MathF.Sin(angle) * scatter,
                                Radius = fRadius,
                                VertexCount = _rng.Next(4, 8),
                                RotationOffset = (float)(_rng.NextDouble() * Math.PI * 2),
                                VelocityX = smaller.VelocityX + MathF.Cos(angle) * 20f,
                                VelocityY = smaller.VelocityY + MathF.Sin(angle) * 20f,
                            });
                        }
                    }
                }
            }

            return fragments;
        }
    }
    ```

    ```csharp
    using BelterLife.Shared.Entities;
    using BelterLife.Simulation.Physics;

    namespace BelterLife.Simulation.Entities;

    /// <summary>
    /// Orchestrates per-tick asteroid physics: Newtonian drift + collision resolution.
    /// Architecture: Simulation/Entities/AsteroidManager.cs (FR7, FR8; FR11–15 in later stories).
    /// </summary>
    public class AsteroidManager
    {
        readonly PhysicsEngine _physicsEngine;
        readonly CollisionResolver _collisionResolver;

        public AsteroidManager(PhysicsEngine physicsEngine, CollisionResolver collisionResolver)
        {
            _physicsEngine = physicsEngine;
            _collisionResolver = collisionResolver;
        }

        /// <summary>
        /// Applies drift to all live asteroids, then resolves collisions.
        /// </summary>
        /// <param name="asteroids">EF-tracked asteroids for this sector (live only; IsDestroyed filtered before passing in).</param>
        /// <param name="deltaSeconds">Tick duration in seconds.</param>
        /// <returns>Newly created fragment asteroids (not yet persisted; Id = 0).</returns>
        public List<Asteroid> ApplyDriftAndResolveCollisions(List<Asteroid> asteroids, float deltaSeconds)
        {
            foreach (var asteroid in asteroids)
            {
                _physicsEngine.ApplyAsteroidDrift(asteroid, deltaSeconds);
            }
            return _collisionResolver.Resolve(asteroids);
        }
    }
    ```

```csharp

    await db.Asteroids.AddRangeAsync(newFragments, cancellationToken);
      ```

```csharp
await db.SaveChangesAsync(cancellationToken);

// 4. Build WorldStateUpdate — fragment IDs are now set
```

### Elastic Collision Formula (derivation reference)

For two circles with mass proxies `m1 = r1²`, `m2 = r2²`:
- `n` = normalised vector from a2 centre to a1 centre
- `dvn` = relative velocity along `n` = `dot(v1 - v2, n)`
- Impulse scalar `j = 2 * dvn / (1/m1 + 1/m2)` = `2 * m1 * m2 * dvn / (m1 + m2)`
- `v1_new = v1 - (j / m1) * n`
- `v2_new = v2 + (j / m2) * n`

If `dvn >= 0` (objects moving apart), **skip** — this prevents double-resolution when two asteroids remain overlapping across multiple ticks (overlap can persist for one tick after fragmentation).
